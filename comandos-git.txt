## Comando git init (iniciando git no repositório)
- O comando git init, inicia um repositório git dentro do projeto, ou seja toda vez que é rodado esse comando o projeto passa a ter o git nele.

## Comando git status (verifica o status do arquivo, pasta etc.)
- Este comando verifica o status de um arquivo e suas mudanças. Além dessa verificação ele analisa em qual estado está esse arquivo.

## Comando git add e git add . (coloca o arquivo em outro estado)
- O comando git add (nome do arquivo), coloca o arquivo em um novo status antes de ser comitado as modificações.

## Comando git diff (analise mais detalhada)
- O comando verifica em qual linha e qual lugar do arquivo foi alteradas.

## Comando git commit -m "mensagem do commit"
- Comando para registrar a alteração e realizar uma documentação do que foi feito.

## Comando git log
- O comando git log, ele basicamente mostrar o histórico de commits, bem como autor e hora do ultimo commit, assim tendo um log do que está acontecendo.

## Comando git restore
- O comando git restore, ele consegue voltar ao status anterior das modificações.

## Comando git restore --staged
- Este comando volta ao estado anterior, depois que você já adicionou na area antes de ser comitado.

ex: depois que foi feito git add . 

## Comando git push
- Envia as modificações para o repositório na nuvem, é usado para enviar todas as modificações feitas no repositório local.

## Comando git pull
- Baixa as informações mais atuais do repositório remoto para repositório local.
